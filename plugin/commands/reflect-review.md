# 리뷰 반영

코드리뷰 코멘트를 검토하고, 수용 여부를 판단하여 코드를 수정한다.

## 실행 방식

이 skill은 worktree(fork)에서 실행된다.
구현 컨텍스트가 보존된 상태에서 실행되어야 한다.

## 절차

1. `.review-artifacts/{branch-name}/review-comments.md`를 읽는다.
2. 각 코멘트에 대해 수용/거부를 판단하고, 판단 근거를 작성한다.
3. 판단 결과를 사용자에게 제시한다.
   - `[p1]` 항목은 기본적으로 수용. 거부 시 명확한 근거 필요.
   - `[p2]` 항목은 판단과 근거를 제시하고 사용자 확인.
   - `[p3]` 항목은 판단을 제시하되 사용자 재량에 맡긴다.
   - `[p4]` 항목은 일괄 수용 또는 무시.
4. 사용자 확인 후 코드를 수정한다.
5. 수정 완료 후 QA를 수행한다:
   - 빌드 확인
   - 테스트 실행
   - 버그 발견 시 수정 루프 (수정 → QA → 재확인)
6. 최종 수정 완료 후, 핵심 변경사항(수정 내역, QA 결과, 버그 수정 커밋)을 요약하여 사용자에게 보고한다.

## 산출물 경로

리뷰 반영 결과는 review-comments.md에 판정을 추가하여 업데이트한다.

```
.review-artifacts/{branch-name}/review-comments.md
```

## 판정 형식

각 코멘트 하단에 판정을 추가한다:

```markdown
### [p1] src/payment/service.ts:42
- 근거: ...
- 내용: ...
- 제안: ...
- **판정: ACCEPT** — 수정 완료. `extractPaymentData()` 함수로 분리.
```

```markdown
### [p2] src/payment/controller.ts:15
- 근거: ...
- 내용: ...
- 제안: ...
- **판정: REJECT** — 현재 구조에서 분리하면 오히려 복잡도가 증가함. design-intent.md 참조.
```
