# 설계의도 문서 작성

현재 대화에서 논의된 설계 내용을 기반으로 설계의도 문서를 작성한다.

## 실행 방식

이 skill은 worktree(fork)에서 실행된다.

## 절차

1. 현재 컨텍스트에서 논의된 요구사항, 설계 결정, 트레이드오프를 분석한다.
2. 아래 구조에 맞춰 `design-intent.md` 초안을 작성한다.
3. 초안을 사용자에게 제시하되, **모호하거나 구체화가 필요한 논의점**을 반드시 함께 정리한다.
   - 예: "A 방식과 B 방식 중 어느 쪽 의도인지?", "이 예외 케이스를 의도적으로 제외한 건지?"
4. 사용자 피드백을 반영하여 확정한다.
5. 확정된 문서를 산출물 경로에 저장한다.

## 산출물 경로

```
.review-artifacts/{branch-name}/design-intent.md
```

`{branch-name}`은 `git branch --show-current`로 resolve한다.

## 문서 구조

```markdown
# 설계의도

## 작업 개요
(이 작업이 무엇을 해결하는가)

## 핵심 설계 결정
(어떤 선택을 했고, 왜 그 선택이 최적인가)

### 결정 1: ...
- 선택지: A vs B
- 선택: A
- 이유: ...

## 의도적으로 제외한 것
(이번 작업 범위에서 의도적으로 빠진 것과 그 이유)

## 주의사항
(이 설계를 구현/리뷰할 때 특히 신경 써야 할 점)
```
